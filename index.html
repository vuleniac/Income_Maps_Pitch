Income Maps

The Income Maps app was envisioned as a tool for identifying attrative locations for retail by mapping the geographic distribution of household income. Even though the original motivation behind the creation of the application was to aid the assesmento of comercial locations, this tool can be used for demographic and ecenomic policy analysis.

The Topologically Integrated Geographic Encoding and Referencing (TIGER) form the US Census Bureu provides spatial extracts containing features that characterize legal and statistical geographic areas. Such feutres include household income distribution which is used by this application.

The application leverages on three data pacakages / sources:

  1. TIGRIS Package (mapping poligones and income distribution statistics)
  2. LEAFLET Package (mapping in html)
  3. Google Maps API (location search)

Income Maps : The Process

The application leverages the Google Maps API in order to find the location specified by the user and returns its geo-spatial coordinates and the geopolitcal description of the location. The process goes as follows:

1. Finding the location

The user inputs a location in the text fields provided (street, city, state, zip). None of the fields is mandatory and if the search has no results it defults to the Empire State Building in New York City. The more information is provided to the search, the more precise the results.

2. Pulling Census Data

Once the location is defined, the information for the county, where that address exists, is pulled from the US Census API. If the search result does not provide a specific county, then the information for the whole state is pulled.

3. Mapping

Once all the geo-incformation is compiled, a heatmap representing the concentration of households in the income baraket selected by the user is plotted. The default income braket is 200,000 USD or more.
